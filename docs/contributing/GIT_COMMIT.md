#  Git Commit 類型對照表

我們遵循 Conventional Commits 規範，請在撰寫 Git Commit 訊息時參考以下類型說明：

| **標籤 (type)** | **用途** | **影響** |
|----------------|---------|---------|
| `feat`  | 新增功能 | 影響程式功能，通常影響版本號 (`minor`) |
| `fix`   | 修正錯誤 | 修正錯誤，通常影響版本號 (`patch`) |
| `docs`  | 文件修改 | 變更 README、註解等，不影響程式碼 |
| `style` | 程式碼格式 | 例如調整縮排、空格、Lint，但不影響功能 |
| `refactor` | 重構 | 重新組織程式碼，不影響功能 |
| `perf`  |效能優化 | 提升執行速度、減少記憶體用量等 |
| `test`  | 測試 | 新增或修改測試 |
| `chore` | 雜項 | 例如 CI/CD 設定、依賴更新，非程式碼變更 |
| `revert` | **還原 commit** | 🔙 例如 `revert: 回復上一次變更` |
